## Integration Check Results

### NEW Findings (this audit)

#### HIGH: Settings RBAC Gap (INT-SETTINGS-RBAC)

`PATCH /api/settings`, `POST /api/settings/logo/:type`, and `DELETE /api/settings/logo/:type` are authenticated but missing `@Roles('admin')` guard. A viewer-role user can mutate business settings.

**Context:** The settings module was added outside the original v1 requirements scope. The original SET-01/SET-02/SET-03 requirements were implemented using Supabase `user_metadata` (no backend endpoint). A separate backend settings module was added later but didn't follow the RBAC pattern established in Phase 6.

**Fix:** Add `@UseGuards(RolesGuard)` + `@Roles('admin')` to write handlers in `apps/backend/src/modules/settings/settings.controller.ts`.

#### MEDIUM: Web Frontend Type Drift (INT-WEB-TYPE-DRIFT)

Three web type interfaces are missing fields that exist in the DB schema and are returned by the API:

- `apps/web/src/types/order.ts` — `OrderItem` missing `subtotal`, `sku`
- `apps/web/src/types/inventory.ts` — `Inventory` missing `minStock`, `maxStock`, `location`
- `apps/web/src/types/product.ts` — `Product` missing `stock`, `imageUrl`

These fields are silently dropped at the TypeScript level. No runtime crashes occur because the UI doesn't reference them, but future features using these fields would fail silently.

#### LOW: Mobile Label Localization (INT-MOBILE-LABELS)

Mobile navigation uses English labels (Dashboard, Articles, Orders, etc.) while the web sidebar is fully localized to Spanish (Panel, Artículos, Pedidos, etc.). This creates a visual inconsistency between platforms.
