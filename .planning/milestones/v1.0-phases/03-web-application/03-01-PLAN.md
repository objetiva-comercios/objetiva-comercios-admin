    Create Supabase client factories following @supabase/ssr patterns:

    1. `src/lib/supabase/client.ts` - Browser client using createBrowserClient():
       - Export createClient() function
       - Use NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY
       - This is for Client Components only

    2. `src/lib/supabase/server.ts` - Server client using createServerClient():
       - Export async createClient() function
       - Import cookies from 'next/headers'
       - Use await cookies() (Next.js 14 async cookies)
       - Configure getAll/setAll cookie handlers
       - This is for Server Components, Server Actions, Route Handlers

    3. `src/lib/supabase/middleware.ts` - Token refresh helper:
       - Export updateSession(request) function
       - Create Supabase client with request/response cookie handlers
       - Call supabase.auth.getUser() to refresh tokens (NOT getSession - security issue)
       - Return the modified response

    4. `src/middleware.ts` - Next.js middleware:
       - Import and call updateSession
       - Configure matcher to exclude static files, _next, images, favicon
       - Matcher pattern: '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)'

    5. Update `.env.example` with required variables:
       - NEXT_PUBLIC_SUPABASE_URL=your-project-url
       - NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key

    IMPORTANT: Use getUser() not getSession() in server code per Supabase security guidelines.
