**Why:** Zod v4.3.6 (installed in @objetiva/types) uses `ZodError.issues` as the array of validation issues. The `.errors` property does not exist on ZodError in Zod v4. Accessing `.errors` returns `undefined`, so `undefined[0]?.message` evaluates to `undefined`, and the nullish coalescing operator always falls through to the generic fallback string. Users never see schema-specific messages like "Invalid email" or "Passwords don't match".

**After the fix:** When a user enters an invalid email, they will see "Invalid email" instead of "Please check your inputs". When passwords do not match, they will see "Passwords don't match" instead of the fallback.

**Do NOT change anything else in these files.** The signupSchema.safeParse() and loginSchema.safeParse() calls are correct — only the error property access needs fixing.
