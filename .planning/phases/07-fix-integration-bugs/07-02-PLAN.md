**middleware.ts** -- Replace the auth redirect logic block (lines 37-56). Keep the Supabase client setup (lines 1-33) and the config/matcher (lines 59-71) unchanged.

Replace the existing redirect logic with deny-by-default:

```typescript
const { pathname } = request.nextUrl

// Public routes that bypass auth
const isPublicRoute =
  pathname.startsWith('/login') ||
  pathname.startsWith('/signup') ||
  pathname.startsWith('/auth/callback')

// Authenticated user on auth page -> redirect to dashboard
if (user && isPublicRoute && !pathname.startsWith('/auth/callback')) {
  return NextResponse.redirect(new URL('/dashboard', request.url))
}

// Unauthenticated user on ANY non-public route -> redirect to login with returnTo
if (!user && !isPublicRoute) {
  const loginUrl = new URL('/login', request.url)
  loginUrl.searchParams.set('returnTo', pathname)
  return NextResponse.redirect(loginUrl)
}

return supabaseResponse
```

Remove the `isProtectedRoute` variable entirely. The `isPublicRoute` variable already exists in the current code -- just remove `isProtectedRoute` and change the second `if` condition from `!user && isProtectedRoute` to `!user && !isPublicRoute`.

**login/page.tsx** -- Update the success redirect to use returnTo:

1. Add `useSearchParams` import: `import { useRouter, useSearchParams } from 'next/navigation'`
2. In the component, add: `const searchParams = useSearchParams()`
3. Change the success redirect (line 66) from:
   ```typescript
   router.push('/dashboard')
   ```
   to:
   ```typescript
   const returnTo = searchParams.get('returnTo') || '/dashboard'
   router.push(returnTo)
   ```
4. Keep `router.refresh()` after the push.

Note: Since LoginPage uses `useSearchParams`, it already has `'use client'` directive. The `useSearchParams` hook works in client components. If Next.js 14 requires a `<Suspense>` boundary for `useSearchParams`, the parent layout should already provide one (the `(auth)` layout). If not, wrap the component's return in `<Suspense>` -- but check first, it's likely fine as-is since the page is already a client component.
