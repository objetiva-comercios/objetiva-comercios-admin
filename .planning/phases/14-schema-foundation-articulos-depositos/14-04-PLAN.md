- All other fields: `z.string().optional().or(z.literal(''))` for text fields
- `precio`, `costo`: `z.string().optional().or(z.literal(''))` — keep as string, numeric returns strings
- JSONB arrays (imagenesProducto, imagenesEtiqueta, etiquetasOcr): skip from form for now (image upload is not in scope, these stay as API-only fields)
- `erpSincronizado`: `z.boolean().optional()`
- `activo`: `z.boolean().optional()`

**Props:** `{ articulo?: Articulo; onSuccess?: () => void; mode: 'create' | 'edit' }`

**Form layout** (per locked decision: "single long scrollable form with visual section dividers"):

Each section: `<div className="space-y-4">` with a section header (`<h3 className="text-sm font-medium text-muted-foreground uppercase tracking-wide">Section Name</h3>`) followed by `<Separator />` and then form fields in a grid.

- codigo (disabled in edit mode — PK cannot change), nombre (full width)
- sku, codigoBarras (2-col grid)
- observaciones (full width, Textarea component)

Section 2 — Propiedades:

- marca, modelo (2-col grid)
- talle, color (2-col grid)
- material, presentacion (2-col grid)
- medida (single)

Section 3 — Precios:

- precio, costo (2-col grid)
- Display hint: "Los precios se guardan con 2 decimales"

Section 4 — Imagenes:

- Show placeholder text: "La gestion de imagenes estara disponible proximamente"
- No form fields (images managed via API, not form)

Section 5 — ERP:

- erpId, erpCodigo (2-col grid)
- erpNombre (full width)
- erpPrecio, erpCosto (2-col grid)
- erpUnidades (single, number input)

Section 6 — Origen:

- originSource, originSyncId (2-col grid)
