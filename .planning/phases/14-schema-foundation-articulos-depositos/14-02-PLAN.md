  <action>
Follow the existing products module pattern exactly. Create:

**articulo-query.dto.ts:** Extend the pattern from product-query.dto.ts:

- `page` (optional, Type(() => Number), default 1)
- `limit` (optional, Type(() => Number), default 20)
- `search` (optional, string) — searches across codigo, nombre, sku, codigoBarras, erpCodigo
- `activo` (optional, Transform to boolean) — true=active only, false=inactive only, omit=all
- `sortBy` (optional, string, default 'createdAt')
- `sortOrder` (optional, 'asc'|'desc', default 'desc')

**create-articulo.dto.ts:** Use class-validator decorators (follow RESEARCH.md code example):

- Required: `codigo` (@IsString @MaxLength(50)), `nombre` (@IsString @MaxLength(255))
- All other fields optional with appropriate validators
- `precio` and `costo` as `@IsOptional() @IsString()` (numeric returns strings)
- JSONB arrays as `@IsOptional() @IsArray()`

**update-articulo.dto.ts:** Same as create but all fields optional (use PartialType or manually make all optional). Exclude `codigo` from update (PK cannot change).

**articulos.service.ts:**

- `findAll(query: ArticuloQueryDto)`: Build WHERE conditions array. If `query.search`, add OR(ILIKE on 5 fields) per RESEARCH.md Pattern 2. If `query.activo !== undefined`, add `eq(articulos.activo, query.activo)`. Apply pagination with `offset = (page-1) * limit`. Return `{ data, meta: { total, page, limit, totalPages } }` matching existing PaginatedResponse shape.
- `findOne(codigo: string)`: Select by PK. Return articulo or null.
- `create(dto: CreateArticuloDto)`: Insert into articulos table. Return created record.
- `update(codigo: string, dto: UpdateArticuloDto)`: Update by PK, set updatedAt to now(). Return updated record.
- `toggleActive(codigo: string)`: Toggle `activo` boolean. Return updated record.

**articulos.controller.ts:**

- `GET /articulos` → findAll(query) — public (auth required by default via JWT guard)
- `GET /articulos/:codigo` → findOne(codigo) — use `@Param('codigo')` (string, no ParseIntPipe)
- `POST /articulos` → create(dto) — `@UseGuards(RolesGuard) @Roles('admin')`
- `PATCH /articulos/:codigo` → update(codigo, dto) — `@UseGuards(RolesGuard) @Roles('admin')`
- `PATCH /articulos/:codigo/toggle` → toggleActive(codigo) — `@UseGuards(RolesGuard) @Roles('admin')`

IMPORTANT: The param is `:codigo` not `:id`. NestJS auto-decodes URL params, so `encodeURIComponent` on client side is enough.

**articulos.module.ts:** Standard NestJS module, export ArticulosService.

**app.module.ts:** Add ArticulosModule to imports array.
