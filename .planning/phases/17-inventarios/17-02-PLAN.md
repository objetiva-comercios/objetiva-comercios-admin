Mirror the depositos module structure exactly. Inject DrizzleService for DB access.

**DTOs (class-validator):**

- `CreateInventarioDto`: nombre (IsString, IsNotEmpty), fecha (IsDateString), depositoId (IsInt), descripcion (IsOptional, IsString)
- `UpdateInventarioDto`: nombre (IsOptional), descripcion (IsOptional) — only name/description editable, not depositoId or estado
- `InventarioQueryDto`: page (IsOptional, IsInt, default 1), limit (IsOptional, IsInt, default 10), estado (IsOptional, IsString), fechaDesde (IsOptional, IsDateString), fechaHasta (IsOptional, IsDateString)
- `CreateInventarioArticuloDto`: articuloCodigo (IsString, IsNotEmpty), cantidadContada (IsOptional, IsInt, default 0), dispositivoId (IsOptional, IsInt), sectorId (IsOptional, IsInt), observaciones (IsOptional, IsString)
- `UpdateInventarioArticuloDto`: cantidadContada (IsOptional, IsInt), dispositivoId (IsOptional, IsInt), sectorId (IsOptional, IsInt), observaciones (IsOptional, IsString)

**Controller endpoints (all prefixed /api/inventarios):**

- `GET /` — list with pagination + filters (estado, fechaDesde, fechaHasta). JWT guard only.
- `GET /:id` — get event detail with count summary (total articulos counted). JWT guard only.
- `POST /` — create event. @Roles('admin'). Returns 201 with created event.
- `PATCH /:id` — update nombre/descripcion. @Roles('admin').
- `PATCH /:id/estado` — transition status. Body: `{ estado: string }`. @Roles('admin'). Validates transition rules.
- `GET /:id/articulos` — get counted articulos with discrepancy data (join with existencias). JWT guard only.
- `POST /:id/articulos` — add articulo to count. @Roles('admin'). Calls assertEventEditable first.
- `PATCH /:id/articulos/:articuloId` — update count. @Roles('admin'). Calls assertEventEditable first. articuloId is the inventarios_articulos.id (integer).
- `DELETE /:id/articulos/:articuloId` — remove from count. @Roles('admin'). Calls assertEventEditable first.

**Service critical methods:**

1. `findAll(query)` — paginated list with optional estado/fecha filters. Include deposito name via JOIN. Return `{ data, meta: { total, page, limit, totalPages } }`.
2. `findOne(id)` — single event with deposito name and count summary (COUNT of inventarios_articulos).
3. `create(dto)` — insert into inventarios, default estado='pendiente'.
4. `update(id, dto)` — update nombre/descripcion only.
5. `transitionEstado(id, newEstado)` — validate transition using `validateTransition()`, update estado + updatedAt. Valid transitions: pendiente->[en_curso, cancelado], en_curso->[finalizado, cancelado], finalizado->[], cancelado->[]. Throw BadRequestException on invalid transition.
6. `assertEventEditable(inventarioId)` — check estado is not 'finalizado' or 'cancelado'. Throw BadRequestException if locked.
7. `getArticulosWithDiscrepancy(inventarioId)` — JOIN inventarios_articulos with articulos (for nombre) and LEFT JOIN existencias (for stock sistema, filtered by the event's depositoId). Use `COALESCE(existencias.cantidad, 0)` for null safety. Calculate diferencia as cantidadContada - stockSistema. Return array of objects with: id, articuloCodigo, articuloNombre, cantidadContada, stockSistema, diferencia, dispositivoId, sectorId, observaciones.
8. `addArticulo(inventarioId, dto)` — assertEventEditable, then insert. Handle unique constraint violation gracefully (return 409 Conflict with "Articulo ya existe en este conteo").
9. `updateArticulo(inventarioId, articuloId, dto)` — assertEventEditable, then update.
10. `removeArticulo(inventarioId, articuloId)` — assertEventEditable, then delete.

Private helper: `validateTransition(current, target): boolean` — transition map per research.

**Module:** Register controller + service, export service.
