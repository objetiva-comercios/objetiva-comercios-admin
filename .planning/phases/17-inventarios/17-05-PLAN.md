**conteo-table.tsx** — client component, core of the counting experience:

Props: `{ inventarioId: number, estado: Inventario['estado'] }`

State management:

- Fetch articulos with discrepancy via client-side call to fetchInventarioArticulos (or pass as server prop and use state)
- isReadOnly: `estado === 'finalizado' || estado === 'cancelado'`
- showOnlyDiscrepancies: boolean toggle (default false)

Table columns:

1. **Articulo** — articuloCodigo + articuloNombre (codigo in muted text-xs above nombre)
2. **Cantidad Contada** — if not readOnly: use InlineEditCell pattern (click to edit, Enter to save, blur to cancel). On save: call updateInventarioArticulo(inventarioId, id, { cantidadContada: newValue }). If readOnly: plain text.
3. **Stock Sistema** — stockSistema (plain text, always read-only)
4. **Diferencia** — diferencia value with color coding:
   - `diferencia === 0`: green text (text-green-600) or neutral
   - `diferencia < 0`: red text (text-red-600) + "Faltante" label or just the negative number
   - `diferencia > 0`: yellow/amber text (text-amber-600) + "Sobrante" label or just the positive number with +
5. **Acciones** — if not readOnly: delete button (trash icon) to remove articulo from count via removeInventarioArticulo. Confirmation before delete.

Above table:

- Discrepancy filter: Toggle/checkbox "Mostrar solo discrepancias" — when active, filter rows where diferencia !== 0
- Summary stats: Total articulos contados, Total con discrepancias (count where diferencia !== 0)

After any mutation (add, update, remove): re-fetch the articulos list to get fresh discrepancy data.

If no articulos in count yet, show empty state: "No hay articulos en este conteo. Usa el buscador para agregar articulos."

**page.tsx** — server component:

```typescript
import { fetchInventario, fetchInventarioArticulos } from '@/lib/api'
import { ConteoTable } from '@/components/inventarios/conteo-table'
import { ArticuloSearch } from '@/components/inventarios/articulo-search'
import { notFound } from 'next/navigation'
import Link from 'next/link'

export default async function ConteoPage({ params }: { params: { id: string } }) {
  const id = Number(params.id)
  if (isNaN(id)) notFound()

  const inventario = await fetchInventario(id)

  const isReadOnly = inventario.estado === 'finalizado' || inventario.estado === 'cancelado'

  return (
    <div className="space-y-4">
      {/* Header with back link and event info */}
      <div>
        <Link href={`/articulos/inventarios/${id}`} className="text-sm text-muted-foreground hover:text-foreground">
          ← Volver al detalle
        </Link>
        <h2 className="text-xl font-bold">{inventario.nombre}</h2>
        <p className="text-sm text-muted-foreground">
          Conteo de inventario — {inventario.depositoNombre} — Estado: {inventario.estado}
        </p>
      </div>

      {/* Search - only if editable */}
      {!isReadOnly && <ArticuloSearch ... />}

      {/* Read-only banner if finalized/cancelled */}
      {isReadOnly && (
        <div className="rounded-md border border-amber-200 bg-amber-50 p-3 text-sm text-amber-800">
          Este inventario está {inventario.estado === 'finalizado' ? 'finalizado' : 'cancelado'}. Los conteos son de solo lectura.
        </div>
      )}

      {/* Counting table */}
      <ConteoTable inventarioId={id} estado={inventario.estado} />
    </div>
  )
}
```

The ConteoTable fetches its own data client-side (useEffect on mount) so it can re-fetch after mutations without full page reload. The ArticuloSearch passes the set of existing codigos from ConteoTable's state to prevent adding duplicates.

Alternative: Make ConteoTable manage both the search and the table internally for simpler state coordination. The search component adds to the count, then ConteoTable re-fetches. Use a shared parent state or callback pattern.

Styling: Tabler aesthetic throughout (text-sm, compact table rows, md border-radius on buttons).
