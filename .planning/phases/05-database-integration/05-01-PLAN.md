1. Create `apps/backend/src/db/index.ts` — the DrizzleService:
   ```typescript
   import { Injectable, Inject } from '@nestjs/common'
   import { drizzle, PostgresJsDatabase } from 'drizzle-orm/postgres-js'
   import postgres from 'postgres'
   import * as schema from './schema'

export const DRIZZLE_CLIENT = 'DRIZZLE_CLIENT'

@Injectable()
export class DrizzleService {
public db: PostgresJsDatabase<typeof schema>

     constructor(@Inject(DRIZZLE_CLIENT) private readonly client: ReturnType<typeof postgres>) {
       this.db = drizzle(client, { schema })
     }

}

````

2. Create `apps/backend/src/db.module.ts` — the @Global NestJS module:
```typescript
import { Global, Module } from '@nestjs/common'
import postgres from 'postgres'
import { DrizzleService, DRIZZLE_CLIENT } from './db'

@Global()
@Module({
  providers: [
    DrizzleService,
    {
      provide: DRIZZLE_CLIENT,
      useFactory: () => postgres(process.env.DATABASE_URL!),
    },
  ],
  exports: [DrizzleService],
})
export class DbModule {}
````

  <name>Task 3: Create seed script and add database CLI commands</name>
  <files>apps/backend/src/db/seed.ts, apps/backend/package.json</files>
  <action>
1. Create `apps/backend/src/db/seed.ts` that adapts existing faker generators to produce Drizzle inserts. The seed script runs standalone via `tsx`:

- First line MUST be `import 'dotenv/config'` (RESEARCH.md Pitfall 3)
- Import `drizzle` from `drizzle-orm/postgres-js`, `postgres` from `postgres`, `sql` from `drizzle-orm`
- Import all table references from `./schema`
- Import existing generator functions from `../data/generators/*.ts` — these return the same shapes as the TypeScript interfaces
- Create standalone postgres client and drizzle instance (NOT using DrizzleService — this runs outside NestJS)

Seed function:
a. TRUNCATE all 8 tables with RESTART IDENTITY CASCADE (single statement, leaf tables first):
`TRUNCATE TABLE purchase_items, sale_items, order_items, inventory, purchases, sales, orders, products RESTART IDENTITY CASCADE`

b. Generate data using existing generators (same volumes per user decision): - `const productsData = generateProducts(500)` - `const ordersData = generateOrders(200, productsData)` - `const inventoryData = generateInventory(productsData)` - `const salesData = generateSales(150, productsData)` - `const purchasesData = generatePurchases(50, productsData)`

c. Insert products first — map generator output to schema columns. The generators return objects with `id` field but we use `serial()` auto-increment, so OMIT the `id` field from inserts and let PostgreSQL generate them. Use `.returning()` to get back inserted rows with their real DB-assigned IDs.

d. Build a mapping from old generator IDs to new DB IDs: `const idMap = new Map<number, number>()` mapping `productsData[i].id -> insertedProducts[i].id`.

e. Insert inventory — use idMap to resolve productId FK references.

f. Insert orders — omit `items` array from order inserts. After each order batch insert, collect the returned IDs and insert order_items using the order IDs and idMap for productId resolution.

g. Insert sales — same pattern as orders (parent insert, then sale_items with resolved FKs).

h. Insert purchases — same pattern (parent insert, then purchase_items with resolved FKs).

i. Call `await client.end()` at the end.

IMPORTANT: For date fields, the generators return ISO 8601 strings. Pass `new Date(dateString)` to Drizzle timestamp columns. For the `items` field on orders/sales/purchases — do NOT try to insert it; it's a virtual field in the generator output, not a DB column.

Console.log progress after each entity type ("Seeding 500 products...", "Seeding 200 orders...", etc.).

2. Add scripts to `apps/backend/package.json`:

   ```json
   "db:generate": "drizzle-kit generate",
   "db:migrate": "drizzle-kit migrate",
   "db:push": "drizzle-kit push",
   "db:seed": "tsx src/db/seed.ts",
   "db:studio": "drizzle-kit studio"
   ```

3. After writing the seed script, generate the initial migration (requires DATABASE_URL in .env):
   ```bash
   cd apps/backend && pnpm db:generate
   ```
   This creates `apps/backend/drizzle/` directory with SQL migration files. These should be committed to git.

NOTE: The seed script must handle the batch insert of potentially 500+ rows. Drizzle's `.insert().values([...])` handles arrays natively. For very large arrays, consider batching in chunks of 100 if needed, but 500 products should be fine in a single insert.
</action>
