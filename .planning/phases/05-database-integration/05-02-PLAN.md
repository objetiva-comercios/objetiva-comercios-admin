  <action>
Rewrite 3 services and update 3 controllers. Every service follows the SAME pattern. Reference RESEARCH.md Pattern 3 (two-query pagination) and Pattern 5 (transactions for multi-table CRUD).

**For ALL services:**

- Remove `import { seedAll }` and `private items: Entity[]` constructor pattern
- Inject `DrizzleService` via constructor: `constructor(private readonly drizzle: DrizzleService)`
- Import schema tables and Drizzle operators: `eq, ilike, and, desc, asc, sql` from `drizzle-orm`
- ALL methods become `async` returning Promises
- Replace `paginate()` helper with two-query pattern (count query + data query)
- Build `PaginatedResponseDto` manually: `new PaginatedResponseDto(data, { total, page, limit, totalPages })`

**products.service.ts:**

- `async findAll(query: ProductQueryDto)`: Build conditions array from query params (search uses `ilike` on name, description, sku with `or()`; category uses `eq`; status uses `eq`; minPrice/maxPrice use `gte`/`lte`). Two queries: count with where, select with where+orderBy+limit+offset. Sort: parse `query.sort` (prefix `-` = desc, else asc) and apply to column. Default sort: `desc(products.createdAt)`.
- `async findOne(id)`: `select().from(products).where(eq(products.id, id))`, return first or null.
- `async getCategories()`: `selectDistinct({ category: products.category }).from(products).orderBy(asc(products.category))`, map to string array.
- `async getStats()`: Use `groupBy` with `count(*)` on status column. Return same shape: `{ total, byStatus: { active, inactive, discontinued } }`.
- NEW `async create(dto)`: `insert(products).values({...dto}).returning()`. Return inserted product.
- NEW `async update(id, dto)`: `update(products).set({...dto, updatedAt: new Date()}).where(eq(products.id, id)).returning()`. Throw NotFoundException if no result.
- NEW `async remove(id)`: `delete(products).where(eq(products.id, id)).returning()`. Throw NotFoundException if no result.

**products.controller.ts:**

- Add imports: `Post, Patch, Delete, Body` from `@nestjs/common`
- Keep existing GET routes, but since service methods are now async, NestJS handles this transparently (no controller changes needed for existing routes)
- Add `@Post() create(@Body() dto: any)` — returns `this.productsService.create(dto)`. Use `any` for the body DTO since we are not building form UIs yet — validation can be added in Phase 6.
- Add `@Patch(':id') update(@Param('id', ParseIntPipe) id: number, @Body() dto: any)` — returns `this.productsService.update(id, dto)`
- Add `@Delete(':id') remove(@Param('id', ParseIntPipe) id: number)` — returns `this.productsService.remove(id)`
- CRITICAL: Keep specific routes (`categories`, `stats`) BEFORE `:id` param route to avoid path conflicts (already established pattern from Phase 2).

**orders.service.ts:**

- `async findAll(query)`: Same two-query pattern. Search on orderNumber, customerName, customerEmail. Filter on status, customerId, minTotal/maxTotal (using `gte`/`lte` on total), startDate/endDate (using `gte`/`lte` on createdAt).
- `async findOne(id)`: TWO queries — fetch order row, then fetch orderItems where orderId matches. Return `{ ...order, items }` to preserve API shape where orders have nested items array.
- `async getStats()`: groupBy status with count. Return `{ total, byStatus: { pending, processing, shipped, delivered, cancelled } }`.
- NEW `async create(dto)`: Use `db.transaction()` — insert order, then insert orderItems with the returned order.id. Return order with items.
- NEW `async update(id, dto)`: Update order fields (not items). Throw NotFoundException if not found.
- NEW `async remove(id)`: Delete order (CASCADE handles orderItems). Throw NotFoundException if not found.

**orders.controller.ts:**

- Add POST, PATCH, DELETE routes same as products pattern.

**inventory.service.ts:**

- `async findAll(query)`: Two-query pattern. Search on productName, sku, location. Filter on status, location, minQuantity/maxQuantity.
- `async findOne(id)`: Simple single-row select.
- `async getStats()`: groupBy status with count. Also query lowStockItems: `select().from(inventory).where(eq(inventory.status, 'low_stock'))`. Return `{ totalItems, byStatus: { in_stock, low_stock, out_of_stock }, lowStockItems }`.
- `async getLowStock()`: `select().from(inventory).where(eq(inventory.status, 'low_stock'))`.
- NEW `async update(id, dto)`: Update inventory quantity/status. No create/delete — inventory is managed via products.

**inventory.controller.ts:**

- Add PATCH route for updating inventory. No POST/DELETE (inventory records are created with products).

IMPORTANT NOTES:

- For the `or()` operator in search: import `or` from `drizzle-orm` and use `or(ilike(col1, pattern), ilike(col2, pattern), ...)`.
- For sorting: parse the `query.sort` string. If it starts with `-`, use `desc(column)`, otherwise `asc(column)`. Map the field name string to the actual schema column reference. If sort is not provided, default to `desc(table.createdAt)`.
- The `PaginatedResponseDto` constructor takes `(data, meta)` — construct meta as `{ total, page, limit, totalPages: Math.ceil(total / limit) }`.

**sales.service.ts:**

  1. Total count + total revenue: `select({ count: count(), totalRevenue: sql<number>'cast(sum(total) as double precision)' }).from(sales)`
  2. By payment method: `groupBy(sales.paymentMethod)` with count
  3. By status: `groupBy(sales.status)` with count
  4. Today's sales: filter `createdAt >= today start`, get count + sum(total)
  5. This week's sales: filter `createdAt >= 7 days ago`, get count + sum(total)
     Return same shape as current mock: `{ totalSales, totalRevenue, averageOrderValue, byPaymentMethod: {cash, card, transfer, credit}, byStatus: {completed, refunded, partial_refund}, todaySales, todayRevenue, thisWeekSales, thisWeekRevenue }`.
     Use `sql` template literals for date comparisons: `sql\`${sales.createdAt} >= ${todayStart}\``or use`gte(sales.createdAt, todayStart)` with a JS Date object for today midnight.

- NEW `async create(dto)`: Transaction — insert sale, then saleItems. Return sale with items.
- NEW `async update(id, dto)`: Update sale fields. Throw NotFoundException if not found.
- NEW `async remove(id)`: Delete sale (CASCADE handles saleItems). Throw NotFoundException if not found.

**sales.controller.ts:**

**purchases.service.ts:**

- `async findAll(query: PurchaseQueryDto)`: Two-query pattern. Search on purchaseNumber, supplierName. Filter on status, supplierId, minTotal/maxTotal, startDate/endDate.
- `async findOne(id)`: TWO queries — fetch purchase row, then purchaseItems. Return `{ ...purchase, items }`.
- `async getStats()`:
  1. Total count + totalSpent: sum of total
  2. Pending orders: filter `status = 'ordered'`, get count + sum(total) for pendingValue
  3. By status: groupBy status with count

NOTE: The `averageOrderValue` in sales stats = totalRevenue / totalSales. Compute in JS after getting the aggregate values to avoid division-by-zero issues.
