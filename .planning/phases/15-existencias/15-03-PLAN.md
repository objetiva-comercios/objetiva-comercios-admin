1. **ExistenciasPorArticulo** (`existencias-por-articulo.tsx`): Client component. Props: `depositos: Deposito[]`, `data: ExistenciaMatrixRow[]`, `isLoading: boolean`, `onStockUpdate: (articuloCodigo: string, depositoId: number, newValue: number) => Promise<void>`, `pageCount: number`, `currentPage: number`, `onPageChange: (page: number) => void`.

This component builds a TanStack Table with dynamic columns:

- **Fixed columns** (frozen on horizontal scroll):
  - `articuloCodigo`: "Codigo", width 120px, font-mono text-sm
  - `articuloNombre`: "Articulo", width 200px
- **Dynamic columns** (one per deposito):
  - Header: deposito.nombre (truncated if long)
  - Cell: renders InlineEditCell with `value={row.stock[deposito.id] ?? 0}`, `onSave` calls `onStockUpdate(row.articuloCodigo, deposito.id, newValue)`
  - Width: ~100px each
- **Total column**:
  - Header: "Total"
  - Cell: renders row.total as bold number, no editing
  - Width: ~80px

**Frozen columns implementation**: Use a raw `<table>` inside a horizontally scrollable div (NOT ServerDataTable — the matrix layout is too different). Use TanStack `useReactTable` directly for column/row management but render with custom table markup.

- Outer container: `<div className="overflow-x-auto border rounded-md">` with max-width from parent
- Left frozen columns: apply `sticky left-0 z-10 bg-background` on codigo column cells/header, `sticky left-[120px] z-10 bg-background` on nombre column cells/header (offset = codigo width). Add `shadow-[2px_0_5px_-2px_rgba(0,0,0,0.1)]` on the last frozen column for visual separation.
- Table styling: `<table className="w-full text-sm">`, `<th>` with `h-8 px-2 text-left font-medium text-muted-foreground bg-muted/50 whitespace-nowrap`, `<td>` with `h-9 px-2 border-b whitespace-nowrap`.
- Pagination: render same pagination controls as ServerDataTable (Previous/Next + page numbers).
- Empty state: "No hay existencias registradas" centered message.
- Loading state: skeleton rows with shimmer.

2. **Transform data for matrix**: In ExistenciasClient, when viewMode is 'por_articulo':
   - Fetch raw existencias via `fetchExistenciasMatrixClient(params)` which returns flat Existencia rows.
   - Transform to `ExistenciaMatrixRow[]`: group by articuloCodigo, build `stock: Record<depositoId, cantidad>`, compute `total = sum(cantidades)`.
   - Store transformed data in state.

3. **Wire into ExistenciasClient** (`existencias-client.tsx`): Replace the placeholder div for 'por_articulo' view with `<ExistenciasPorArticulo>`. Pass depositos list, transformed matrix data, pagination state, and onStockUpdate callback. The onStockUpdate callback should: call `updateExistenciaClient`, update local matrix data optimistically, refetch KPI.

   When viewMode changes to 'por_articulo', fetch matrix data. When it changes to 'por_deposito', fetch by-deposito data. Each view manages its own pagination independently.
