    2. Default export function receiving `{ error: Error & { digest?: string }, reset: () => void }`
    3. `useEffect` that logs the error to console on mount: `console.error('[SectionName Error]', error)` -- never expose error details to users

    The component structure for each file:
    ```
    'use client'
    import { useEffect } from 'react'
    import { Button } from '@/components/ui/button'
    import { AlertCircle } from 'lucide-react'

    export default function [Section]Error({ error, reset }) {
      useEffect(() => { console.error('[Section Error]', error) }, [error])
      return (
        <div centered with p-8 gap-4>
          <AlertCircle h-10 w-10 text-destructive />
          <h3>Unable to load [section name]</h3>
          <p text-muted-foreground>Something went wrong. Please try again.</p>
          <Button variant="outline" onClick={reset}>Try again</Button>
        </div>
      )
    }
    ```

    IMPORTANT: Do NOT try/catch in Server Components (page.tsx files). Let fetch errors propagate naturally so error.tsx catches them. This is how Next.js error boundaries work.
