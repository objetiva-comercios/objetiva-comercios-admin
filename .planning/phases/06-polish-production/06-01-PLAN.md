<objective>
Implement backend RBAC with Admin/Viewer role enforcement on all write endpoints.

Purpose: AUTH-06 requires a user roles and permissions system. The critical bug is that `JwtAuthGuard.extractUser()` currently reads `payload.role` (always `"authenticated"` -- the Postgres database role) instead of `payload.app_metadata?.role` (the actual custom app role). This fix is the foundation; then `@Roles()` decorator + `RolesGuard` enforce Admin-only access on POST/PATCH/DELETE endpoints across all 5 domain controllers.

Output: Working RBAC system where Admin users have full API access and Viewer users can only read data. All GET endpoints remain open to both roles.
</objective>
